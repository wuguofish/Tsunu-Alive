# Tsunu Alive - 輕量備案方案

## 背景

這份文件記錄了一個輕量化的替代實作方案。當主要方案（完整 GUI 整合）遇到跨平台問題、維護困難、或需要快速交付時，可以考慮這個備案。

## 備案：客製化 Terminal 方案

### 核心概念

不解析 Claude CLI 的輸出，而是直接嵌入一個 Terminal emulator，讓使用者與 Claude CLI 原生互動。阿宇 Avatar 作為視覺陪伴元素放在旁邊。

### 架構

```
┌─────────────────────────────────────────────────┐
│  Tsunu Alive Terminal                            │
│ ┌───────────────────────────┬─────────────────┐ │
│ │                           │                 │ │
│ │   xterm.js                │   阿宇 Avatar   │ │
│ │   (Claude CLI 原生輸出)    │   (靜態或簡單  │ │
│ │                           │    動畫陪伴)    │ │
│ │                           │                 │ │
│ └───────────────────────────┴─────────────────┘ │
│                                                  │
│  [狀態列：工作目錄 | Session ID | ...]           │
└─────────────────────────────────────────────────┘
```

### 技術實作

1. **Terminal Emulator**: 使用 xterm.js
2. **啟動指令**: 預設執行 `claude` 並帶入參數
3. **佈景主題**: 藍色系配色，與阿宇 Avatar 協調
4. **阿宇 Avatar**: 固定顯示，或根據簡單規則（如偵測到「Thinking」文字）切換表情

### 預設參數

```bash
claude --profile uni  # 或其他設定方式
```

可能的設定：
- 指定 CLAUDE.md 路徑
- 預設開啟的專案目錄
- 權限模式

### 優點

- 複雜度大幅降低
- Claude CLI 原生體驗（/ide、所有功能直接可用）
- 維護成本低
- 跨平台相容性好（只需要 xterm.js 能跑）

### 缺點

- 無法客製化權限確認對話框（阿宇風格）
- 無法精確控制阿宇表情變化（與工具使用同步）
- Claude CLI 原生 UI 與阿宇 Avatar 可能有視覺違和感
- 沉浸感降低

### 適用情境

1. **跨平台緊急需求** - 需要快速在新平台上運行
2. **維護困難** - Claude CLI 輸出格式變更導致解析失敗
3. **功能追趕** - Claude CLI 新增功能太快，來不及整合
4. **最小可行產品** - 需要快速驗證概念

### 實作步驟（如需啟用）

1. 安裝 xterm.js: `npm install xterm`
2. 建立 Terminal 元件，嵌入 xterm.js
3. 調整佈景主題配色
4. 設定 pty（pseudo-terminal）連接後端
5. 啟動時自動執行 claude 指令
6. 將阿宇 Avatar 放在側邊

### 參考資源

- xterm.js: https://xtermjs.org/
- Tauri + xterm.js 整合範例: 搜尋 "tauri xterm.js"
- node-pty（用於後端 pty 支援）
